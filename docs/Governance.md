# Project Governance

## Overview

The eBPF for Windows Sample Program project provides educational examples of eBPF programming on Windows.

## Roles

### Contributor
Anyone can contribute by reporting issues, submitting pull requests, and participating in discussions.

### Maintainer
**Personnel:** @Alan-Jowett (initial maintainer)

**Responsibilities:**
- Review and merge pull requests
- Manage issues and project direction
- Maintain coding standards and quality

## Project Goals

1. **Educational Value**: Provide clear, well-documented eBPF examples
2. **Quality**: Maintain high standards for code and documentation
3. **Accessibility**: Make eBPF development approachable for newcomers

### Sample Program Criteria
- Demonstrate specific eBPF concepts
- Include comprehensive documentation
- Follow project coding standards
- Work with current eBPF for Windows releases

## Pull Request Process

**Requirements:**
- At least one maintainer approval
- Follow coding conventions in [Development Guide](DevelopmentGuide.md)
- Include appropriate documentation
- Resolve all review comments

**Review Criteria:**
- Code correctness and educational value
- Adequate documentation and comments
- Follows project style guide
- Includes tests or validation when applicable

## Communication

- **GitHub Issues**: Bug reports, feature requests, questions
- **Pull Request Comments**: Code review and technical discussion

## Code of Conduct

This project follows the [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/). Report violations to [opencode@microsoft.com](mailto:opencode@microsoft.com).

## Contact

For questions about governance:
- Create a GitHub issue
- Contact maintainers directly
- Email [opencode@microsoft.com](mailto:opencode@microsoft.com) for code of conduct issues
